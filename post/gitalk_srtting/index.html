Gridea中Gitalk配置出现的坑

今天在配置Gitalk中，评论页面一直没有显示出来，总是出现问题。网上查了很多的办法，由于自己是自定义域名，所以和官方提供的文档还是不一样的。

声明：本文的方法可能仅适用于自定义域名

Gitalk 是一个基于 GitHub Issue 和 Preact 开发的评论插件。如果你博客使用的是 Gitalk 的话，建议你可以选择 Gitalk 来作为评论插件。

你可以按照 Gitalk 的文档 进行相关字段配置，或者按照下面的步骤操作。
首先需要有一个Github OAuth Application，点击这里进行创建，字段填写可以参考下图（使用github.io的域名建议参考官方文档）

备注：在此处我自己重新创建了一个仓库用于存评论的，自己可以使用原来存代码的仓库或者重新创建一个仓库用于存放评论。
创建完成后将 Client ID 和 Client Secret复制，保存到本地

如果忘记复制点击此处找到自己刚刚创建的OAuth Apps
接下来就可以去应用里面填写对应的字段信息，开启Gitalk评论。

这样Gitalk评论插件就设置好了，可以通过预览查看是否文章页显示评论模块，但是实际上并不会显示任何评论信息，需要将博客同步到线上才可以正常显示，因为Github OAuth 设置的域名为线上的域名

在配置中出现的坑

我一开始按照官方文档进行配置的使用，总是出现以下情况

  1. Error: Not Found
    一般这个问题的出现是Gridea中仓库（repo）的名称填写错误，正确填写仓库（repo）字段即可。
  2. 未找到相关的Issues进行评论
    这里有两种情况，配置完成后出现这种情况，点击使用GitHub登陆跳转到Github登陆后会自动创建Issues。还有另外一种情况是跳转到个人博客主页面，然后问题依旧存在的，请前往此处查看自己的Homepage URL或者Authorization callback URL是否填写错误。
  • Homepage URL填写仓库地址
  • Authorization callback URL填写回调地址，填写自己的域名或者**.github.io

  1. Error: Validation Failed
